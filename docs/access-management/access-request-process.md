# Access Request Process

## Objective
Provide a controlled and auditable process for granting access to systems, shared resources, and groups.

## Required Information
- Requester name
- Target user name
- System or resource requested
- Business justification
- Manager approval
- Access duration if temporary

## Standard Process
1. Validate request completeness
2. Confirm approval authority
3. Check whether access requested is role-appropriate
4. Implement least-privilege access
5. Record fulfillment in ticket
6. Notify requester when completed
7. Set review or expiration date for temporary access

## Do Not Fulfill Without
- documented approval
- clear business reason
- enough information to identify the correct resource
