# Documentation Standards

## Objective
Keep knowledge articles clear, consistent, and maintainable.

## Standards
- Use descriptive file names
- Start each article with purpose and scope
- Use numbered steps for procedures
- Include escalation guidance when relevant
- Avoid environment-specific secrets or sensitive information
- Update revision history for major changes
- Prefer plain language over jargon where possible

## Writing Style
- Clear and direct
- Action-oriented
- Consistent headings
- Avoid unnecessary filler text
