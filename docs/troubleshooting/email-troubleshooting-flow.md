# Email Troubleshooting Flow

## Common Symptoms
- Cannot send email
- Cannot receive email
- Outlook will not open
- Mailbox not updating
- Shared mailbox missing
- Authentication prompts continue

## Triage Questions
1. Is the issue affecting one user or multiple users?
2. Is the issue limited to Outlook desktop, webmail, or mobile?
3. Did the problem start today or after a recent password/device change?
4. Are there any known service outages?

## Troubleshooting Steps
1. Confirm network connectivity
2. Verify the user can sign in to webmail
3. Check whether the mailbox is licensed and active
4. Confirm password was not recently changed without reauthentication
5. Remove and re-add the account only if lower-risk steps fail
6. Check Outlook profile status and cached mode behavior
7. Validate shared mailbox permissions if that is the issue
8. Escalate if issue appears service-side or tenant-wide

## Escalate When
- Multiple users are affected
- Mail flow appears delayed across departments
- Licensing and permissions appear correct but access still fails
- Error points to tenant, server, or service health issue
