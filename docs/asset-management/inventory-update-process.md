# Inventory Update Process

## Objective
Keep asset records accurate through the full device lifecycle.

## Update Events
- New asset received
- Asset assigned
- Asset transferred
- Asset returned
- Asset sent for repair
- Asset retired or disposed

## Required Fields
- Asset tag
- Serial number
- Device type and model
- Assigned user or location
- Status
- Date of last update
- Notes

## Standards
- Update records as part of the workflow, not later when possible.
- Use consistent status values.
- Record exceptions and missing assets clearly.
