# Password Reset Guide

## Objective
Provide a consistent and secure process for assisting users with password resets.

## Standard Process
1. Verify the user's identity using approved methods.
2. Confirm whether self-service reset is available.
3. Reset password using approved tool if needed.
4. Require the user to change the temporary password at next sign-in if policy requires.
5. Confirm MFA still works after reset.
6. Ask user to update credentials on mobile devices or VPN client if needed.

## Common Issues After Reset
- Cached credentials in Outlook or mobile mail
- VPN still using old password
- Account lockout from repeated bad password attempts on another device

## Escalate When
- Identity cannot be verified
- Account lockout persists after reset
- Self-service reset portal fails repeatedly
